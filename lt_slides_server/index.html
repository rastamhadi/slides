reveal-js

まずスライドを作るツールとしては reveal-js を愛用しています。見栄えもいいんだし、完全にブラウザーベースで、WEB エンジニアの僕にとっては扱いやすいです。なお Speaker Notes 機能まで付いていますので、ナレーションは何も考えずに、事前に書いたものを棒読みできます。短所をひとつ挙げるとしたら、生 HTML のスライドはやっぱりボイラープレートが多いです。そこで…

reveal-ck

http://jedcn.github.io/reveal-ck/

… reveal-ck という gem が登場します。Github Flavored Markdown で書いたスライドを元に、reveal-js 用 HTML を生成してくれます。しかも絵文字にまで対応しています。

ホスティング

次の課題は、スライドを会議室のディスプレイと Google Hangouts に届けること。ラスタムはノートパソコンを持ってないし、持っていたとしても大量の HTML と Javascript と CSS ファイルをコピペするのが嫌です。そこで自分のデスクトップから公開することにしました。この仕組みを立ち上げるには、まずインストールが…

ホスティング手順

　
　

OSX はインストール不要！

　
　
　
　
　
　

…一切ないです！OSX にはデフォルトで Apache が同梱されているから！

ホスティング手順

$ mkdir ~/Sites

次は Sites フォルダーを作ります。OSX はデフォルトでこのフォルダーを WEBROOT として認識しますので、Finder で開いてみれば、Safari の羅針盤アイコンが付いているのが見受けられます。

ホスティング手順

$ ln -s /path/to/reveal.js/slides ~/Sites/hoge

Sites フォルダーの中から、公開したいフォルダーへのシンボリックリンクを張ります。

ホスティング手順

　

$ vi /etc/apache2/users/$(whoami).conf

<Directory "/Users/あなたのユーザ名/Sites/">
  AllowOverride All
  Options Indexes MultiViews FollowSymLinks
  Require all granted
</Directory>

　
　
　
　
　
　

Apache にシンボリックリンクを認識させるように、ユーザ設定をこうします。

ホスティング手順

　

$ vi /etc/apache2/extra/httpd-userdir.conf

Include /private/etc/apache2/users/*.conf

　
　
　
　
　
　
　
　

ユーザ設定の Include 文がコメントアウトされている場合は、アンコメントする必要があります。有効にするために。

ホスティング手順

　

$ vi /etc/apache2/httpd.conf

LoadModule authz_host_module libexec/apache2/mod_authz_host.so
LoadModule authz_core_module libexec/apache2/mod_authz_core.so
LoadModule userdir_module libexec/apache2/mod_userdir.so
LoadModule vhost_alias_module libexec/apache2/mod_vhost_alias.so
Include /private/etc/apache2/extra/httpd-userdir.conf
Include /private/etc/apache2/extra/httpd-vhosts.conf

　
　
　
　
　
　

あと httpd.conf のこの Directive もアンコメントします。何の Directive なのかは、聞かないでください。Google 先生の指示どおりに書いただけですから。

ホスティング手順

　

$ sudo apachectl restart

　
　
　
　
　
　
　
　

最後に Apache を再起動して、新しい設定を読み込ませます。

ホスティング手順

http://あなたのIP/~あなたのユーザ名/シンボリックリンク

うまくいった場合は、IP で見れるようになりました。他の人のマシンからでも。
